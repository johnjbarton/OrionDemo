Orion Web-Based Development

Who

  • Do you develop applications to run in a browser?
  • Do you use a browser to develop those applications?
  • Orion: a Web based system for developing Web based systems.
  • Orion is an AJAX Web App from the github era

    • What

  • JavaScript editor written in JavaScript
  • JavaScript/HTML/CSS based client
  • Java (Eclipse) Web server with REST API
  • git for version control and backup

    • Installation Options

    1. Don't: Just use orionhub.org
    2. localhost: Install server, then open http://localhost:8080
    3. Department or group server, etc.

    Since git provides distributed backup, you don't really need to a fancy server system

    User Interface Overview

    • Login (localhost: you can use a simple login)
    • Home Page
    • Repositories: Use http git URL in Clone Repository
    • git status: Dedicate a browser tab to this one
    • Navigator/Project tree: Do It Right (Click)
    • Editor: feels like Eclipse on a starvation diet
    • Plugins, eg Nonymous, can extend/change UI

    Plugins

    • URLs for registration in Plugins page
    • Implemented as Web Pages installed in iframes,
    • eg NonymousPlugin.html

    Workflow

    • git status page to fetch/merge from remote repo.
    • project navigator page to open test page
    • project navigator page to open file
    • edit
    • git status page to commit, push changes.

    Future Directions for Orion

    • 0.4, 0.5, ...
    • Orion Editor built into Firefox
    • Editor and REST API used in Purple
    • Investigating small integration with Chrome Devtools
